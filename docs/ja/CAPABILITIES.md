# CBIO 機能ガバナンス

本ドキュメントは Claw-biometric (CBIO) プロトコルの**機能マトリクス**とガバナンスモデルを定義します。ガバナンスは**ID 自律**と**委任機能**によって実現されます。

---

## 1. ロールより ID

CBIO v1.0 では、「Admin」や「User」のような固定「ロール」はありません。すべてのエンティティは独自の暗号鍵と vault を持つファーストクラスの**ID**です。

異なる動作は、ID ハンドル (CbioAgent) に特定の**機能**を委任することで実現されます。

| エンティティ | ハンドル | スコープ | 主な責務 |
| :--- | :--- | :--- | :--- |
| **Authority** | `CbioIdentity` | **ルート** | ルート設定、発行、グローバル監査。 |
| **Delegate** | `CbioAgent` | **スコープ** | 自動化、ネットワークインタラクション、タスク固有ロジック。 |

---

## 2. 機能マトリクス

`CbioAgent` ハンドルは、署名付きプロトコル証明書または明示的なランタイム割り当てにより、以下の機能を動的に持つことができます：

| 機能 | 関連メソッド | リスクレベル | 説明 |
| :--- | :--- | :--- | :--- |
| `vault:fetch` | `fetchWithAuth()` | 低 | 保存済みシークレットでネットワークリクエストを認証。 |
| `vault:list` | `listSecretNames()` | 低 | このハンドルが利用可能なシークレット名を列挙。 |
| `vault:acquire` | `fetchAndAddSecret()` | 中 | 新しい第三者認証情報を取得・プロビジョニング。 |
| `admin:secrets` | `agent.admin.xxx()` | 高 | シークレットの完全管理（追加/削除/更新）。 |
| `admin:issue` | `issueManagedAgent()` | **クリティカル** | 新しい子 ID を発行・ガバナンス。 |

---

## 3. ガバナンスループ

SDK は、これらの委任機能のライフサイクル管理のための組み込みメカニズムを提供します。

### 3.1 イントロスペクション（`can` メソッド）
エージェントは**事前チェック**により安定性と決定論的計画を確保できます：
- `agent.can(capability)`：`boolean` を返す。実行前に戦略を適応するために使用。
- `agent.permissions`：現在強制されている制限の読み取り専用ビュー。

### 3.2 オーソリティ自動同期
ID が有効な **IssuedAgentIdentity** 証明書を保持している場合、SDK は証明書の `capabilities` フィールドからランタイム権限を自動導出します。「法的付与」（証明書）と「物理ガード」（ランタイムロック）の間の手動同期は不要です。

### 3.3 失効
オーソリティは、ネットワーク全体で委任 ID を恒久的に失効させることができます：
- `identity.admin.revokeManagedAgent(pubKey)`：署名付き **RevocationRecord** を発行。

---

## 4. ベストプラクティス

1. **実行前に検証**：自律エージェントは `can()` で自身の境界を照会し、優雅なフォールバックを提供すべきです。
2. **最小権限**：特定のサブプロセスに必要な原子機能のみを付与。
3. **監査可視性**：`getManagedAgentCapabilities()` を定期的に使用してアクティブな委任を監査。
